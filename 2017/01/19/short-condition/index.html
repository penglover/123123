if문 대신 삼항연산자와 짧은조건문을 써봅시다~(javascript)

2017-01-19
  • javascript
  • web

삼항 연산자

보통 if문과 switch문으로 코딩을 많이 합니다.
삼항 연산자 조건문은 이들보다 간단하고 직관적으로 코딩이 가능합니다.
사용법은 아래와 같습니다.
(불표현식)?(참일 때 실행하는 문장):(거짓일 때 실행하는 문장)
{codeblock samhang.js}
var input = prompt(“숫자를 입력해주세요.”,””);
var number = Number(input);

(number >= 0) ? alert(“양수입니다.”) : alert(“음수입니다.”);
{endcodeblock}
자, 위의 문장은 어떤 의미를 가지고 있을까요?
입력값을 받은뒤 입력값이 0이상이면 “양수입니다.”를 출력합니다.
0미만이라면 “음수입니다.”를 출력하겠지요.
코드가 매우 가독성이 좋고 짧지요?

짧은 조건문

짧은 조건문은 논리 연산자의 특성을 조건문으로 사용한 것입니다.
true || alert(“실행안됨”);
위의 문장은 실행이 될까요?
자바스크립트는 true까지만 읽고 참임이 확실하기 때문에 추가 연산을 진행하지 않습니다.
짧은 조건문은 이런점을 이용한 조건문입니다.


위의 코드는 어떨까요?
입력값을 받고 만약 그것이 짝수라면 2번째 줄에서 input % 2 == 0이 참이므로 더이상 진행하지 않습니다.
하지만 3번째 줄은 and 연산이므로 뒤까지 봐야 전체 문장이 true인지 false인지를 알수 있습니다.
따라서 뒤의 문장이 읽혀 “짝수입니다.”가 출력되게 됩니다.

둘다 많이 쓰이나요?

삼항 연산자는 짧은 코드를 선호하는 개발자들이 굉장히 많이 사용합니다.
저도 선호합니다.
짧은 조건문은 코드를 봤을 때 한눈에 코드를 파악하기 어려워 많이 사용되지는 않습니다.
